# 제과제빵 기능사 실기 연습 — 작업 가이드 (후속 모델용)

이 문서는 남은 작업을 다른 모델(예: Opus)이 이어서 진행할 수 있도록 쓴 플레이북이다.
사용자가 "계속 진행해"라고 하면 이 문서대로 진행한다.

## 프로젝트 구성

- `제과제빵 실기 연습.dc.html` — 메인 학습앱. 목록/랜덤출제/상세(시험지·공정 카드·퀴즈·합격 공략 4탭). 데이터는 `items-data.json`을 fetch해서 렌더. **이 파일은 거의 손댈 필요 없음** — 데이터만 채우면 자동으로 상세 탭이 열린다.
- `items-data.json` — 40품목 데이터. 현재 **파운드케이크(pound-cake)만 detailed:true**, 나머지 39개는 기본 정보만 있음.
- `모형 카드 - 파운드케이크.dc.html` — A안 인쇄 키트 (A4 4쪽: 재료 카드 9장 + 도구 카드 10장 / 공정 카드 8장+정답표 / 계량대 보드+오븐·체크포인트 / 공정 순서 보드).
- `모형 카드 - 파운드케이크 B안.dc.html` — B안 실습 보드 (8개 스테이션, 각 단계에 맞는 재료/도구 카드를 트레이에 올리는 시뮬레이션 + 정답표). A안에서 오린 카드를 사용.
- `doc-page.js` — 인쇄용 starter component. 수정 금지.

## 남은 작업 (사용자 승인 순서대로)

1. **39품목 상세 데이터 채우기** (아래 스키마). 5품목 정도씩 배치로 `run_script`로 JSON에 병합 → 중간중간 앱에서 확인.
2. 품목별 **모형 카드 A안/B안** 생성. 파운드케이크 파일을 패턴으로 복제 후 데이터·아이콘 교체. 40품목×2파일은 많으므로 **사용자에게 우선순위(자주 출제/어려운 품목부터 등)를 먼저 물어볼 것**.

## 데이터 스키마 (items-data.json 의 detailed 품목)

파운드케이크 항목을 정확한 예시로 참고. 필드:

- `detailed: true` 로 변경
- `temp` 반죽온도 (예: "27℃"; 슈처럼 없으면 생략 → 앱이 칩을 숨김)
- `gravity` 비중 (제과만, 해당 시)
- `requirements[]` — 큐넷 공개문제 요구사항을 **원문 형식대로** ("배합표의 각 재료를 계량하여 재료별로 진열하시오 (N분)." 부터 시작. 제빵은 "반죽은 스트레이트법으로 제조하시오" / 비상법 조치사항 등)
- `ingredients[]` — `{n:재료명, r:"비율", w:"무게"}`. 공개문제 배합표 그대로. 정확성이 가장 중요 — 확신 없으면 웹서치로 최신 공개문제 확인.
- `totalR`, `totalW` — 합계 (문자열, 천단위 콤마)
- `steps[]` — `{t:제목, b:배지(시간/온도/비율, 없으면 ""), d:설명}` 7~9단계. 제빵 공통 흐름: 계량→반죽(최종단계·반죽온도)→1차발효→분할·둥글리기→중간발효→성형→팬닝→2차발효→굽기. 제과는 제법별로 다름.
- `deductions[]` 6~8개, `disqualify[]` 5개 내외 (공통: 미완성/제법 위반/미숙·탄 제품/위생 심각 위반/부정행위 + 품목 고유: 예. 롤케이크 시트 찢어짐), `tips[]` 5~6개 — 실전 꿀팁, 계절 팁 포함.
- 모형 카드를 만든 품목은 `printKit: "모형 카드 - <품목명>.dc.html"` 추가 → 앱 상세에 버튼 자동 표시.

말투: 기존 데이터와 동일하게 간결한 한국어, ~다/명사형 종결. 이모지 금지.

## 모형 카드 키트 만드는 법

A안 파일을 복제해 수정하는 방식. 각 품목마다:
- 재료 카드 = 그 품목 배합표 재료 (icon 매핑 `ingIcons`에 새 재료 아이콘 추가 필요: 강력분, 이스트, 제빵개량제, 쇼트닝, 물, 우유, 앙금, 계피가루 등)
- 도구 카드 = 품목별 도구 (제빵: 저울, 볼, 스크레이퍼, 밀대, 발효실, 식빵팬/평철판, 온도계, 붓, 가위, 행주 등)
- 공정 카드 = steps 그대로 (번호 없음 + 접는 정답표)
- 보드: 계량대(재료 카드와 같은 3열·같은 높이) / 공정 순서 보드(공정 카드와 같은 2열·같은 높이) / 오븐·체크포인트
- B안: 스테이션 defs 배열만 교체 (단계별 사용 재료/도구 매핑 + ask 퀴즈 질문 + 정답표)

### 아이콘 스타일 (중요 — 통일할 것)
- viewBox 0 0 64 64, stroke #6B4423, stroke-width 2.2, 파스텔 채움 (#F5E7CE 크림, #F4B8C1 핑크, #F7D473 버터, #DFF0E4 민트, #EAF2FB 하늘, #E8B04B 빵 표면, #C89B6B 나무)
- 귀여운 스타일: 주요 재료/완성품에는 eyesSmile()+blush() 헬퍼로 표정 추가
- 재료끼리 확실히 구분되는 사물로 그리기 (예: 유화제=펌프병, 달걀=표정 있는 달걀)

## 기술적 함정 (이미 겪은 버그 — 반복 금지)

1. **`<img src="{{홀}}">` 금지** — 브라우저 프리로드 스캐너가 리터럴 홀을 fetch해 콘솔 에러. 반드시 `<span style="background-image:url('{{홀}}')">` + `sc-if iconsReady` 래핑.
2. **data URL에 `;utf8` 금지** — 스타일 파서가 세미콜론에서 자름. `'data:image/svg+xml,' + encodeURIComponent(...)` 형식만 사용.
3. **카드와 보드 칸 크기 일치** — 카드는 `height`(min-height 아님)+`box-sizing:border-box`+`overflow:hidden` 고정, 슬롯도 같은 `height`·같은 열 수·같은 gap. 슬롯 그리드를 패딩 있는 박스로 감싸지 말 것(폭이 어긋남).
4. 페이지 분리는 `<div style="break-before:page"></div>`, 각 카드/보드에 `break-inside:avoid`.
5. 앱 데이터 갱신은 `items-data.json`만 수정 (run_script + JSON.parse/stringify 병합 권장, 손 편집 시 유효 JSON 유지).
6. 스타일은 전부 인라인. 시험지 모형은 'Nanum Myeongjo', 본문은 'Noto Sans KR'.

## 검증

- 각 배치 후 `ready_for_verification`으로 앱 열어 확인 (해당 품목 상세 4탭 + 퀴즈 동작).
- 키트는 인쇄 페이지 수/break 확인.

### 남은/개선 여지
- 파운드케이크 전용 A/B 파일을 범용 엔진으로 통합할지(현재는 병존). 통합 시 items-data의 pound-cake.printKit 를 `모형 카드.dc.html?item=pound-cake` 로 바꾸고 전용 파일 2개 삭제.
- 배합표 수치는 표준 공개문제 기준으로 작성 — 시행처 최신 공개문제와 대조 검수 권장.
- 상세 데이터 수정은 `items-data.json`, 카드 도구/스테이션은 `kit-data.json`만 고치면 됨(엔진·앱 자동 반영).
